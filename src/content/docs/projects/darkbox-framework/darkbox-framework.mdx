## Context

DarkBoxFramework is a Python research framework for automated object recognition. It explores a practical question from computer vision work: instead of choosing image-processing steps by hand, can a system search for a better sequence of transforms and classifier settings by treating the pipeline itself as something to optimize?

## The Problem

Image-recognition experiments can become a manual feature-engineering loop: choose transforms, tune parameters, train a classifier, compare scores, and repeat. This project tried to automate that loop by representing the pipeline itself as the thing being searched.

## Constraints

- **Huge search space.** Combining transform order, transform parameters, and classifier settings creates many candidate pipelines.
- **Heterogeneous operations.** Image filters, feature descriptors, morphology, thresholding, and classifiers need a common optimizer-facing interface.
- **Repeatable evaluation.** Every candidate must be scored consistently across a training image set.
- **Research visibility.** Experiment metrics and artifacts need to be recorded for later inspection.
- **Archival status.** The project reflects experimental research code from 2014 to 2016 rather than a current production package.

## The Design

The project models an image-processing pipeline as DNA. A solution is encoded as an ordered list of transforms and parameters, then evaluated by applying that pipeline across a training image set, extracting features, and scoring a scikit-learn classifier. The best candidates are then mutated, re-scored, and retained through an evolutionary search loop.

The main optimization path combines Artificial Immune System ideas with simulated annealing. The `AISSA` implementation creates and scores a population of candidate pipelines, sorts candidates by fitness, applies rank-influenced mutation, and uses simulated annealing as part of replacement decisions. The wider optimizer folder also includes implementations or scaffolding for genetic algorithms, particle swarm optimization, ant colony optimization, artificial bee colony optimization, and genetic programming.

### Transform Registry

The transform layer wraps a large set of image-processing operations behind a registry-style architecture. The repository includes 56 transform classes spanning blurs, edge detectors, corner detection, HOG, local binary patterns, Gabor filters, morphology, thresholding, rank filters, and exposure adjustments. This made the search space broad enough to test many feature-engineering paths without hard-coding one fixed pipeline.

### Evaluation And Artifacts

Evaluation was built around scikit-learn classifiers and repeatable experiment output. The framework records metrics such as accuracy, F1, precision, recall, and Matthews correlation, uses joblib to parallelize image processing across the image set, and can persist results through flat files, MySQL-backed records, or HTTP posting. The repository also contains 33 serialized `.pkl` learner artifacts from previous runs.

The project is intentionally archival: it reflects experimental research code from 2014 to 2016 rather than a production package. As a portfolio project, it shows early hands-on work with computer vision, automated feature engineering, evolutionary algorithms, experiment tracking, and the engineering tradeoffs involved in turning a search idea into a runnable framework.
