*Author · 2016*

**At a glance**

- 87 learner shell classes across regression, classification, and clustering model families
- 15 parameter enum types for bounded, categorical, gaussian, dynamic, constrained, and unconstrained hyperparameters
- 139 run-output files and 51 analysis artifacts from evolutionary model-search experiments

scikit-evolution is a Python framework for treating machine-learning model configuration as an optimization problem. It wraps scikit-learn-style estimators in "shell learner" adapters, defines their searchable hyperparameter spaces, and uses evolutionary search to discover model and parameter combinations that perform well on a given dataset.

The main search path is an Artificial Immune System optimizer. It creates a population of candidate `ShellLearnerSolution` objects, scores each candidate with scikit-learn `cross_val_score`, sorts by fitness, mutates candidates with rank-influenced intensity, avoids duplicate configurations, and keeps the best-performing candidates across generations. The optimizer layer also includes hill climbing, random search, and modules for genetic algorithms, simulated annealing, particle swarm optimization, tabu search, and evolution strategies.

The learner abstraction is the core of the project. The repository contains 87 learner shell classes across regression, classification, and clustering model families, including linear models, trees, ensembles, SVMs, nearest-neighbour methods, neural networks, Gaussian processes, kernel ridge models, naive Bayes, TensorFlow/skflow wrappers, and lightning wrappers. Helpers group those learners by problem type and data-size suitability so experiments can search a selected pool rather than a single estimator.

The hyperparameter system gives each shell a common vocabulary for randomization and mutation. The `EParameter` enum defines 15 parameter categories, including integer, real, string, boolean, array, map, category, bounded, mixed, dynamic, gaussian, Dirichlet, constrained, unconstrained, and kernel values. Bounds, movement rules, randomizers, and mutators sit behind that vocabulary so the optimizer can mutate very different model settings through one interface.

The framework also includes experiment infrastructure around the search loop. It supports custom scoring such as MAPE, regression and classification problem scripts, domain-specific normalizers for load-forecasting style work, MySQL query builders and schema scripts for persisted results, file-output fallbacks, and a timeout path for evaluations that might hang.

The repository preserves evidence of real experimentation: 139 run-output files, 51 analysis artifacts, SQL scripts for querying best learners and simulation results, and 19 test files covering mutation, randomization, parameter bounds, metrics, classifiers, regressors, data handling, preprocessing, SQL queries, GPU paths, and database connectivity.

The project is intentionally archival. As a portfolio project, it shows early hands-on work with AutoML-style search, model abstraction, hyperparameter typing, evolutionary algorithms, experiment persistence, and the practical messiness of making many heterogeneous machine-learning libraries behave through one optimizer-facing contract.
